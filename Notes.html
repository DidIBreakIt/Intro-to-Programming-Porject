Webpages, Documents, and Structures

Lesson1: The Basics of the Web and HTML

The Web
- A collection of HTML documents, links between the HTML documents create its web-like nature.
- It contains a wide array of files, including plain text, images, audio, and video.
- HTTP is the main protocol of the web
HTML (HyperText Markup Language)
- It is made up of text content, markup (what it looks like), references to other documents (i.e. videos, images), and links to other pages.
HTML markup
- Made up of tags, in the format of <name>contents</name>.
- The opening tag, contents, and closing tag, as a whole, are refered to as an element.
- The b tag makes the contents appear bold.
- The em tag makes the contents appear in italics.
- Forgetting to write in the closing tag can lead to disasterous results because the computer does not know inherently where the closing tag should be.
- Not all tags require a closing tag, such as <br> and <img>. In those instances, the closing tag is referred to as a void tag.
HTML attributes
- Written in the format <tag attribute = "value">contents</tag>.
- Tags can have multiple attributes.
- An exampe of a tag that uses an attribuite is the anchor tag, <a href = "url">content</a>. Anchors are for making links.
- Another example is the image tag, which is written out as <img src = "url" alt = "text"> (there is no closing tag). The src is the address of where the image is located, and the alt is the text that gets displayed when the image does not load.
White Space
- If you enter text on multiple lines, it is still rendered in HTML as one line.
- The break tag, <br>, creates a line break. It is a void tag.
- The paragraph tag, <p>content</p>, also creates a break by making a block around its contents.
Inline vs. Block
- Inline tag affects the contents in the line of HTML.
- Examples of inline elements include <b>, <em>, <img>, and <br>.
- Block tag creates an invisible box around its contents. The box can have height and width.
- Examples of block elements include <p>.
HTML Documents
- The Doctype describes the which kind of HTML this is.
- The opening and closing HTML tag surround the rest of the document.
- The head includes metadata, CSS and Java, and the title of the page, which appears on the top of the browser window.
- The body is the actual contents of the document.

Lesson 2: Creating a Structured Document with HTML

- HTML helps design the structure of the page. CSS determines the style of the page. Java lets users interact with the page.

- Every HTML page has a branching, tree-like structure, otherwise known as the DOM(Document Object Model).

- HTML classifies page content.

- CSS changes how elements look on the page.

- Every element on an HTML page is a rectangle. CSS allows other shapes to be displayed on the page.

- The page is actually like a grid, and boxes can placed and rearranged on the grid.

Lesson 3: Adding CSS Style to HTML Structure

CSS
- Stands for Cascading Style Sheets
- Cascading means that the style from the most specific is applied.
Inheritance
- Any elements enclosed in another element take on the property values of the enclosing element.
CSS Syntax
-To select the element class you want to apply style, type either the tag name or the .class before a pair of curly brackets, {}.
- To select with attribute you want to change, type attribute:value inside the curly brackets. The attributes are separated by a semi-colon.
- Examples: p{font-size:medium}, .descrpition{color:white; background-color:red} - All modifiable attributes and possible values can be found in the CSS Reference
HTML Elements
- They are boxes, and each box has four components: contents, padding, border, and margin.
- Divs are block elements, so they take up the entire width of the page.
-The CSS rule {display:flex} allows block elements to be displayed side by side.
Code, Test, Refine
1. Look for natural boxes.
2. Look for repeated styles and semantic elements.
3. Write your HTML.
4. Apply styles (from biggest to smallest).
5. Fix things.

Lesson 2: Variables and Strings

Variables
- A variable is a name that refers to an expression.
-It is defined by an assignment statement, name = expression.
- Variables can have their value changed. - They can be used in their own assignment statement, i.e. days = days - 1.
- In programming, the equals symbol means assignment not equivalency.
- Convention says that variables should start with a lower case letter.
Strings
- A string is a sequence of characters surrounded by quotes.
- It can be either single or double quotes, but it must start and end with the same type of quote.
- Strings can be combined together.
- Strings cannot be added to numbers, however they can be multiplied by numbers.
Indexing Strings
- Each character of a string is assigned a value starting with 0 for the first character of a string and going up sequencially from there.
- The character, in a certain position of a string, can be evaluated by the expression: <string>[<expression>], where the expression in the brackets evaluates to a number.
- I.e. 'udacity'[0]='u' or 'udacity'[1+1]='a'
- A negative number inside the brackets evaluates the from the end, i.e. 'udacity'[-1]='y' or 'udacity'[-3]='i'.
Selecting Subsequences
- A subsequence of a string can be found by the expression: <string>[<expression>:<expression>], where both expressions inside the brackets evaluate to numbers.
- The subquence string starts at the position of the first expression in the bracket and ends at the second expression in the bracket minus one.
- I.e. 'udacity'[1:4]='dac' or 'udacity'[2:-2]='aci'
- Leaving the right side open will return a subsequence from the first number to the end of the string, i.e. 'udacity'[3:]='city'.
- Leaving the left side open with return a subsequence from the beginning of the string til the number on the left minus 1, i.e. 'udacity'[:4]='udac'.
- Leaving both sides open will return the entire string.
Finding Strings in Strings
- <string>.find(<string>) will return the position of the first occurence of the substring in the string.
- I.e. 'I am happy.'.find('happy')=5.
- Another parameter can be added to the find procedure to see where the substring occurs after a given position.
- I.e. 'Happy happy happy'.find('hap', 7)=12
